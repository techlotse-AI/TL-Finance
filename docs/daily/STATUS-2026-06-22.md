## Resolved today — the D4 net-worth schema question

Yesterday's report held D4 back pending one question: *do manual cash/property balances need a new Prisma table?* Inspecting `prisma/schema.prisma` answers it:

- **`Account` has no balance field** — balances are derived (transactions / pockets / statement imports). There is **no model for manually-entered assets** (property, vehicle, valuables) or stand-alone liabilities beyond what is derivable.
- Therefore the scope splits cleanly:
  - **D4-a (1 day, no migration):** net worth from *already-modelled, derivable* sources — holdings market value (`lib/optimize/holdings.ts`) + pension vehicles + derived account balances − credit-card/debt balances (`lib/optimize/debt.ts`), aggregated into one reporting currency via `ExchangeRate`. Pure `lib/optimize/net-worth.ts` + golden tests + `api/optimize/net-worth/route.ts` + read-only workspace panel. **Honest limitation: excludes property/other real-world assets**, so it should be labelled "financial net worth (modelled accounts only)".
  - **D4-b (follow-up, needs migration):** additive `ManualAsset` / `ManualLiability` table (name, type, value, currency, valuation date) to capture property/vehicles and complete the picture. This is what makes net worth genuinely useful for the Swiss-first audience, but it is more than one clean day.
